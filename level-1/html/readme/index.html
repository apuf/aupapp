    Оглавление

    HTML

    HTML - HyperText Markup Language язык разметки гипертекста

    Любой сайт в интернете основывается на html.

    Для понимания html нужно понять что такое языки разметки в принципе.

    Кодировки

    Как мы знаем из прошлых разделов - компьютеры обрабатывают и хранят дынные в виде битов, двоичных чисел со значениями 1 и 0 8 битов составляют байт. Итого возможно 256 комбинаций. Если назначить каждой возможной комбинации двоичных чисел в байте букву то мы получим набор букв и спец символов. таблица по которой мы конвертируем символы из байтов называется кодировкой. Байтовые кодировки могут хранить в себе 255 символов. Примеры таких кодировок ASCII, cp1251, cp1250, КОИ-8 и др.

    Таким образом мы можем получать текст на одном из языков. windows до сих пор использует такие кодировки для каждого языка например cp1251 - русская кодировка.

    Такой подход ограничивает текст документа набором из 255 символов на 1 язык. Для английского и русского этих символов хватает а вот для языков вроде китайского уже нет. Для того чтобы создать более объемные кодировки текста были созданы кодировки из нескольких байтов например utf-8, utf-16, utf-32 и прочие unicode кодировки. В таких кодировках хватает закодировать все символы большинства языков (в том числе и мертвых - например скандинавские руны) и еще остается на различные знаки вроде копирайта, эмодзи, математических, статистических и прочих научных символов.

    Разметка

    После того как у нас есть текстовые файлы в той или иной кодировке нам необходимо реализовать такие простые для текста вещи вроде указания на кегль и размер шрифта, выделение курсивом или жирным шрифтом, построение списков, таблиц, выделение цветов или фон текста. Для решения подобных проблем были созданы языки разметки TeX и GML (General Markup Language).

    TeX используется для верстки типографии и научных документов, GML помимо верстки являлся еще и машиночитаемым форматом, позволяющим удобно обрабатывать текст в различных программах. На основе GMS был создан SGML и уже на его основе были созданы XML и HTML.

    Для разметки текста в TeX и HTML/XML используются разные наборы символов. Однако бывает ситуация когда нам нужно напечатать в тексте символ который используется для разметки - тогда нам необходимо так называемое экранирование.

    Экранирование позволяет с помощью набора символов отобразить символы которые зарезервированы для разметки документа

    основы XML и HTML

    Для того чтобы отобразить какой либо элемент в XML/HTML используются теги.

    теги

    Тег это набор символов заключенных в знаки < и >

    Пример: <документ>

    тег должен состоять из двух частей - первая открывает тег а вторая закрывает.

    Пример: <документ></документ>

    Для закрытия тега используются тот же имя тега но с косой чертой после знака <

    Если тег только один тогда можно использовать сокращенную запись

    Например: <документ/>

    Теги можно помещать один в другой, однако всегда должен быть один корневой тег который описывает сам документ.

    например:

    <персона>
    <имя>Василий</имя>
    <фамилия>Пупкин</фамилия>    
</персона>

    В данном случае <персона> это корневой тег и он должен быть только один.

    Теги не должны пересекаться, если мы отрыли тег и еще один тег тогда мы сперва закрываем последний открытый тег а потом уже остальные.

    Пример того как делать не нужно:

    <персона>
    <имя>Василий<фамилия>
    Пупкин</имя></фамилия>    
</персона>

    атрибуты

    Любой тег может содержать атрибуты, они обозначается словом внутри открывающего тега символом = после него и значением взятым в одинарные или двойные кавычки

    Пример:

    <персона пол="мужской" возвраст='28'>
    <имя>Василий</имя>
    <фамилия>Пупкин</фамилия>    
</персона>

    еще раз об экранировании

    Если нам необходимо в xml/html указать символы который используется для разметки - < > ' " & нам необходимо указать специальный код для этих символов.

    • & = &amp;
    • " = &quot;
    • ' = &apos;
    • < = &lt;
    • > = &gt;

    Данный способ позволяет помещать эти символы в текст html/xml документов

    <документ>
    5 &lt; 8
    &apos;ТОО&apos; &quot;Рога&amp;Копыта&quot; 
</документ>

    типы документов

    Стандарты html и xml могут изменяться со временем и для того чтобы различать где какой тип документа в самое начало помещается комментарий с типом документа.

    html5:

    <!DOCTYPE html>
<html>
...
</html>

    html4:

    <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
<html>
...
</html>

    xml:

    <?xml version="1.0" encoding="utf-8"?>
<document>
...
</document>

    xml+html4 (xhtml 1.1):

    <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN" "http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd">
<html>
...
</html>

    разница между html/xhtml/xml

    xml позволяет определять любые теги однако: - все теги должны находиться внутри корневого тега - все атрибуты тегов обязательно должны содержать значение. - Теги не должны пересекаться. - Для документов должна быть указана кодировка. - Документы могут иметь пространства имен и схемы которые используются для проверки документа. - Пространства имен позволяют использовать несколько схем, а схемы позволяют дополнить правила проверки своими вариантами.

    Если документ создан с ошибками или не соответствует схемам тогда он считается ошибочным и не будет открыт.

    xml создан для обмена между различными программами, хранения конфигураций и какое-то время использовался для создания новостных лент.

    В html: - все теги и их атрибуты строго определены и вы не можете просто так добавлять новые теги. - В отличии от xml в html браузер может сам исправлять ошибки документа и продолжать работу даже если документ не идеален.

    Проверка документа называется валидацией а прошедший проверку документ называется валидным.

    xhtml это формат который позволяет использовать html и xml одновременно и обладает свойствами html4 и xml.

    xhtml и xml считаются устаревшими форматами, xml постепенно вытесняется json a xhtml не прижился из за своей строгости к ошибкам. html же продолжает использоваться и модифицироваться, последняя версия на данный момент - html 5.

    изучаем html5

    В html5 документ состоит из корневого тега <html></html> В его параметры можно внедрять xml схемы например open graph.

    Далее документ может иметь шапку (заголовки) и тело документа.

    В документе могут содержаться комментарии которые никак ни влияют на сам документ но их могут прочитать другие люди и вы сами в исходном коде страницы.

    Комментарии обрамляются особым тегом <!-- -->

    Пример

    <!-- Это комментарий -->

    HEAD

    Заголовок документа определяться тегом <head/> Данный тег содержит служебные заголовки для браузера.

    Title

    Тег <title> используется для описания заголовка вкладки браузера В него нет смысла помещать другие теги - только текст который вы хотите увидеть на вкладке.

    Создайте html файл со следующим содержанием

    <!DOCTYPE html>
<html>
    <head>
        <title>Your Site Name</title>        
    </head>
    <body>

    </body>
</html>

    Если открыть этот файл в браузере мы увидим пустую страницу, но имя вкладки будет Your Site Name

    Если заменить Your Site Name на Сайт Ивана Ивановича тогда по идее если открыть его вкладка будет называться Сайт Ивана Ивановича. Но если мы работаем в блокноте windows тогда есть шанс что мы увидим ���� ����� ��������� или ???? ????? ?????????

    Для того чтобы устранить эту досадную оплошность нам нужно понять в чем проблема. В блокноте windows при сохранении мы выбираем кодировку ANSI или cp1251 однако большинство браузеров понимают сайты в виде кодировки utf-8 если кодировка не указана явно.

    Мы можем указать кодировку явно либо переконвертировать его в utf-8.

    Для удобства работы с html лучше использовать специальный редактор, который не только поможет узнать кодировку и переконвертировать файл но и будет давать подсказки для html. Таких редакторов огромное множество. На правах рекламы рекомендую купить JetBrains intelliJIdea/Webstorm/PhpStorm если вы не желаете покупать такие мощные но дорогие инструменты то есть альтернативы попроще - notepad++, Geany(который я использую на ноуте который не тянет Idea), SublimeText2 или Github Atom.

    Meta

    Тэг <meta /> используется для того чтобы передать браузеру ту или иную информацию о странице.

    Для того чтобы указать кодировку используется атрибут charset

    Для utf-8 например 

    <meta charset="utf-8">

    или для русской window

    <meta charset="windows-1251">

    Для старых браузеров есть и другая форма записи кодировки из html4

     <meta http-equiv="content-type" content="text/html; charset=utf-8" />

     <meta http-equiv="content-type" content="text/html; charset=windows-1251" />

    Однако лучше всего использовать utf-8 во всех файлах сайта. Так как в эту кодировку уже входят все символы языков мира и у вас не будет проблем со вставкой цитаты на сайт например на греческом языке или математической формулы если оно вам понадобиться.

    Помимо кодировки в <meta /> может включать и другую информацию, например описание сайта для поисковых систем, ключевые слова для поиска сайта, информацию для различных соцсетей например facebook или vk.

    <html prefix="og: http://ogp.me/ns#">
    <head>
        <title>ТОО Рога и копыта</title>        
        <meta name="Description" content="ТОО Рога и копыта - лучшие пластиковые окна всех цветов и размеров под заказ">
        <meta name="Keywords" content="пластиковые окна, под ключ, на заказ">
        <meta content="no-cache" http-equiv="Cache-Control">
        <meta content="ru" http-equiv="Content-Language">
        <meta name="viewport" content="width=device-width, initial-scale=1">
        <meta property="og:type" content="website" />
        <meta name="author" content="Иванов Иван Иванович" />
        <meta charset="utf-8" http-equiv="content-type" content="text/html; charset=utf-8" />
        <meta property="og:url" content="окна.ru" />
        <meta property="og:title" content="Посмотри на мир через новое окно" />
        <meta property="og:image" content="https://окна.ru/logo.jpeg" />
        <meta property="og:description" content="Для клиентов из Facebook у нас скидка 25%" />
        <meta property="og:site_name" content="ТОО Рога и копыта" />
    </head>

</html>

    Мы еще не начали делать сайта и у нас нет ни единой строчки текста а мы уже передаем огромное количество информации о нашем сайте.

    Link

    Иногда нам нужно добавить ссылку на иконку сайта в самой вкладке или указать таблицу стилей - для этого используется тег link

    Примеры

    <!-- Иконка для вкладки -->
<link rel="shortcut icon" href="/images/favicon.png" type="image/x-icon">
<!-- Основная таблица стилей -->
<link rel="stylesheet" href="/css/main.css" type="text/css" media="all">
    Другие теги в Head

    Head может содержать и другие теги, например <script>, <style> или <base>

    <script> и <style> мы рассмотрим в других частях этой книги.

    Теги в body или что нам стоит сайт построить

    В этом разделе огромное количество тегов и их атрибутов, некоторые уходят в историю, некоторые не меняются уже долгое время и некоторые появились относительно недавно.

    Нужно понимать что html это "Живой" стандарт который постоянно меняться и следует следить за новостями чтобы не пропустить ничего важного.

    Для полного понимания html лучше пройти отдельный курс. В этой книги будут только самые основы без которых нежелательно будет приступать к php. Который и есть наша основная цель.
